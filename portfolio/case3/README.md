# I-FOODS storefront prototype

Первая версия сайта собрана как статический storefront-прототип без сборки и внешних зависимостей.

## Что уже есть

- главная витрина в логике дизайн-документа;
- hero, категории, хиты, каталог, featured-product блок и секция бренда;
- реальные товары, собранные из [еда.docx](./документы/еда.docx);
- локальная корзина и guest-checkout прототип с выбором способа получения;
- расчёт доставки для курьера и самовывоза внутри одного checkout-потока;
- подтверждение заказа в демо-режиме без внешнего платёжного шлюза.

## Структура

- [index.html](./index.html) — разметка страницы
- [styles.css](./styles.css) — дизайн-система и адаптивная вёрстка
- [script.js](./script.js) — данные товаров, фильтры, featured-panel и cart state
- [design_document_ifoods_project.md](./документы/design_document_ifoods_project.md) — проектная рамка
- [изображения](./изображения) — локальные ассеты

## Локальный запуск

Самый простой вариант:

```powershell
python -m http.server 8080
```

После этого откройте:

```text
http://127.0.0.1:8080
```

Если не нужен локальный сервер, `index.html` можно открыть напрямую в браузере, но сервер предпочтительнее.

## Сохранение правок админа

- каталог и карточки товаров сохраняются в `data/catalog.json` через `api/catalog.php`;
- загруженные фото попадают в `uploads/`;
- для работы на хостинге нужен **PHP** и права на запись в папки `data/` и `uploads/`.

## Что ещё не закрыто

- реальные бизнес-правила по зонам, тарифам и SLA доставки после согласования;
- интеграция с платёжным провайдером и отправка заказа во внешнюю систему;
- полные карточки состава, КБЖУ и аллергенов;
- юридические страницы и контакты бренда.
